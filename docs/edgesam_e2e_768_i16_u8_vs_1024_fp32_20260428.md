# EdgeSAM 768 int16/uint8 E2E Circle Check

Date: 2026-04-28

## Compared Setups

| Setup | Input size | Encoder | Decoder |
| --- | ---: | --- | --- |
| Baseline | 1024 | FP32 Circle | FP32 Circle |
| Test | 768 | int16 Circle | uint8 Circle |

The 768 encoder and decoder were exported from the EdgeSAM checkpoint and compiled with Samsung ONE. The encoder used `quantized_dtype=int16`; the decoder used `quantized_dtype=uint8`. Both quantized models kept float32 model inputs and outputs.

## COCO Val Result

Evaluation command used COCO val images, center point prompt, one mask per image, and `mask_threshold=0.0`.

| Metric | 1024 FP32 e2e | 768 int16+uint8 e2e | Delta |
| --- | ---: | ---: | ---: |
| selected mIoU | 0.688731 | 0.543963 | -0.144768 |
| oracle mIoU | 0.820410 | 0.722011 | -0.098399 |

Run size: 5 images x 1 mask.

The selected mIoU drop is 14.48 points, about 21.0% relative to the 1024 FP32 e2e baseline. The oracle mIoU drop is 9.84 points, so the loss is not only score ranking noise; candidate masks are also worse.

One image dominated the drop:

| Image | selected mIoU delta | oracle mIoU delta |
| --- | ---: | ---: |
| `000000472375.jpg` | -0.450259 | -0.339706 |
